# Phase 04.1 Plan 01: /model Command Summary

**One-liner:** `/model` command wired to OpenCode PATCH /config and GET /config/providers — users can list and switch AI models from Telegram.

## What Was Built

### Task 1: OpenCode config API client (`src/opencode/config.ts`)

Three API functions following the same fetch pattern as `session.ts`:

- `patchConfig(baseUrl, model)` — PATCH /config with `{ model }` body; throws `unknown_model` on 400, generic error on other failures
- `getConfigProviders(baseUrl)` — GET /config/providers; returns `Record<string, ProviderInfo>`
- `getConfig(baseUrl)` — GET /config; returns `{ model?: string }`

8 unit tests (TDD red→green) covering success and all error paths.

### Task 2: `/model` command handler + bot wiring

`makeCmdModelHandler(registry, openCodeUrl)` factory with two paths:

**No-arg path** (`/model`):
1. Fetches current model via `getConfig` (fallback: "unknown" on error)
2. Fetches provider map via `getConfigProviders`
3. Formats reply: current model on line 1, then "Available models:" with providers sorted alphabetically by providerID
4. On providers fetch error: replies `❌ Could not fetch available models. Is OpenCode running?`

**Switch path** (`/model anthropic/claude-sonnet-4`):
1. Calls `patchConfig` with trimmed (not lowercased) arg
2. Success: `✅ Model switched to {arg} (global — affects all sessions).`
3. `unknown_model` error: `❌ Unknown model "{arg}". Run /model to see available models.`
4. Other errors: `❌ Could not switch model. Is OpenCode running?`

**Wiring:**
- `bot.command("model", makeCmdModelHandler(...))` added to `src/bot/index.ts`
- `{ command: "model", description: "..." }` added to `setMyCommands` in `src/main.ts`
- `/model [name] — Switch active AI model or list available models` added to `cmd-help.ts`

7 unit tests (TDD red→green) covering all paths.
